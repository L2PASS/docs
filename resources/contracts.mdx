Markdown tables can look "weird" or inconsistent in size because of how browsers handle **table cell distribution** and **intrinsic image spacing**. When you have long, unbreakable strings like full contract addresses in one column and small icons in another, the browser stretches columns unevenly.

Additionally, standard Markdown doesn't include "padding" controls, so it defaults to the styling of your specific environment (like a website's CSS), which often adds extra whitespace to rows.

### Why it looks "weird":

1. **Vertical Misalignment:** By default, images sit on the "baseline," making them look like they are floating too high compared to the text.
2. **Column Stretching:** The full `0x...` addresses are very wide, forcing the "Contract Address" column to take up most of the screen and squishing the other columns.
3. **Default Padding:** Many Markdown renderers add `12px` to `16px` of padding per cell, which looks "huge" when using small 24px icons.

## Security and Audits

Security is the foundation of the L2Pass protocol. Our contracts are built using industry-standard libraries such as **OpenZeppelin** and feature several layers of protection:

* 
**Non-Custodial:** The protocol never takes custody of user assets; all swaps and yield positions are managed via smart contracts.


* 
**Pausability:** In the event of extreme market volatility or external threats, the `Pausable` module allows the team to protect user funds by temporarily halting deposits.


* 
**Reentrancy Guard:** All critical functions are protected against recursive call attacks.
